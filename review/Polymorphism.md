# Polymorphism Review

- 評価: `Good`

## Good: 無理な継承階層や不自然な共通化は見当たらない
- 優先度: Low
- 対象: `include/*`, `src/*`
- 詳細と根拠:
  - `include` / `src` 配下を確認した範囲では、公開 API も内部実装も継承ベースではなく、値オブジェクトと合成で構成されています。
  - `rg "class .*:|struct .*:" include src` の結果でも、継承を伴うクラス定義は見当たりませんでした。
  - このライブラリの責務は「音声再生の薄いラッパ」であり、現状の規模で無理にポリモーフィズムを入れない判断は妥当です。
- 改善案/方針:
  - 今後バックエンド差し替えやストリーミング実装を導入する段階までは、このまま合成ベースで進める方が安全です。

## Good: Handle ベースの API は is-a より role ベースで整理されている
- 優先度: Low
- 対象: `include/Handle.hpp`, `include/PlaybackHandle.hpp`, `include/TrackHandle.hpp`
- 詳細と根拠:
  - `Handle`, `PlaybackHandle`, `TrackHandle` はそれぞれ「ロード済み音源」「再生インスタンス」「ミキサートラック」という別役割を素直に分離しています。
  - 共通基底クラスにまとめていないため、`Bird` / `Penguin` 的な誤った is-a 関係は発生していません。
- 改善案/方針:
  - この役割分離は維持してよいです。共通化したくなっても、安易に抽象基底を追加しない方がよいです。
