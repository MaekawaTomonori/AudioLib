# AntiMagicNumbers Review

- 評価: `Warning`

## Good: 一部の定数は `constexpr` 名称付きで管理できている
- 優先度: Low
- 対象: `src/Loader.cpp`
- 詳細と根拠:
  - `src/Loader.cpp:11-12` の `kOutputChannels`, `kOutputSampleRate` のように、少なくとも意味のある名前は付いています。
  - 生の数値をそのまま散らしているよりは一段よいです。
- 改善案/方針:
  - 命名付き定数に寄せる姿勢は継続してください。

## Warning: 出力チャンネル数とサンプルレートが固定値で、ライブラリ利用者が調整できない
- 優先度: High
- 対象: `src/Loader.cpp`
- 詳細と根拠:
  - `src/Loader.cpp:11-16` でデコード設定を `2ch / 44100Hz` に固定しています。
  - これは単なる「数字がむき出しかどうか」の問題ではなく、意味の重い運用パラメータが実装に埋め込まれている点が問題です。
  - 音声ライブラリとしては、元ソース保持、デバイスに追従、あるいは初期化時設定に委ねる選択肢が欲しいです。
- 改善案/方針:
  - `Initialize()` 時の設定項目として外出しするか、最低でも `AudioConfig` に集約してください。
  - 「なぜ 44100Hz / stereo 固定なのか」をコメントで正当化できないなら、固定値で持つべきではありません。

## Warning: デフォルト値の意味が API 仕様として明文化されていない
- 優先度: Medium
- 対象: `include/Handle.hpp`
- 詳細と根拠:
  - `Handle::Play()` の `1.f`, `1.f`, `0.f`, `false` は一般的ではあるものの、範囲制約や異常値時の扱いが API 契約として曖昧です。
  - 音量やパンはまだ理解しやすいですが、ピッチ範囲とクランプ有無は実装依存です。
- 改善案/方針:
  - `AudioDefaults` や API コメントで、既定値と受理範囲、クランプ方針を明示してください。
